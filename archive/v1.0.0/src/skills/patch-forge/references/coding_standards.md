# ธรรมนูญการเขียนโค้ดสำหรับกองทัพ Oracle (Coding Principles & Standards) 🛡️🌊

เพื่อให้โค้ดของ **Pulse CLI** และโครงการในเครือ มีความเสถียร (Stable), ดูแลรักษาง่าย (Maintainable), และไม่เกิด Regression แม้จะมีการ Patch ซ้อนไปซ้อนมา ทีมงานต้องยึดถือหลักเกณฑ์ 5 ข้อดังนี้:

---

## 1. ยึดหลัก Open-Closed Principle (OCP)
**"โค้ดควรเปิดสำหรับการขยาย (Extension) แต่ปิดสำหรับการแก้ไข (Modification)"**
*   **แนวทาง**: พยายามเขียนโค้ดใหม่เป็นโมดูลหรือปลั๊กอินแยกออกมา แทนการรื้อลอจิกเดิมในไฟล์หลัก
*   **เป้าหมาย**: เพื่อให้โค้ดเก่าที่เสถียรแล้วไม่มีวันพังจากการเพิ่มฟีเจอร์ใหม่

## 2. ควบคุมสถานะด้วย Feature Flags
**"ห้ามลบโค้ดทิ้งจากประวัติเด็ดขาด หากฟีเจอร์นั้นอาจถูกเอากลับมาใช้ใหม่"**
*   **แนวทาง**: ย้ายลอจิกไปซ่อนไว้หลังเงื่อนไขสวิตช์ (Feature Toggle) ใน `config.ts`
*   **การใช้งาน**: ปิดฟีเจอร์ = `false`, เปิดฟีเจอร์ = `true` เพื่อรักษา Git History ให้สะอาดและย้อนกลับได้ง่าย

## 3. ทำ "สัญญา" ด้วย Type และ Interface (Strict Typing)
**"ห้ามใช้ `any` เด็ดขาด ทุกอย่างต้องมีนิยามที่ชัดเจน"**
*   **แนวทาง**: นิยามโครงสร้างข้อมูลผ่าน `interface` หรือ `type` ใน `config.ts` หรือ SDK
*   **เป้าหมาย**: เพื่อให้ TypeScript Compiler ช่วยดักจับ Error ตั้งแต่ตอนเขียน (Build-time validation)

## 4. กฎเหล็ก "Write Test Once, Run Forever" (Automated Testing)
**"สิ่งเดียวที่ยืนยันว่าโค้ดไม่เสีย คือ Test"**
*   **แนวทาง**: เมื่อมีการแก้จุดเสี่ยงหรือจุดเชื่อมต่อข้ามไฟล์ ต้องเขียน **Integration Test** คลุมไว้เสมอ
*   **การบังคับใช้**: สคริปต์การ Patch ต้องพ่วงคำสั่ง `bun test` ท้ายสุดเสมอ

## 5. นโยบายเลิกใช้งานโค้ดเก่า (Deprecation Policy)
**"อย่าลบโค้ดในทันที ให้เวลาทีมปรับตัว"**
*   **แนวทาง**: ใส่ Tag `@deprecated` และเขียน Log แจ้งเตือนไว้ล่วงหน้า 1-2 เวอร์ชัน ก่อนจะลบจริงใน Major Update

---

## 📝 Checklist ก่อนส่งงาน (Sacred Anchor Verification)
1. [ ] การแก้ครั้งนี้ใช้การเพิ่มโมดูลใหม่ หรือใช้วิธีรื้อลอจิกเดิม? (พยายามเลือกเพิ่มโมดูล)
2. [ ] โครงสร้าง Interface ใน `config.ts` ถูกประกาศ Type ครบถ้วน ไม่ใช้ `any` ใช่ไหม?
3. [ ] มีการอัปเดตไฟล์ `patch.manifest.json` เพื่อบันทึกประวัติแล้วหรือยัง?
4. [ ] รัน `bun test` ผ่านหมด 100% หรือไม่?
