# Functional Requirement: Individual Skill Refresh (FR-14)

## 1. Overview
เพิ่มความสามารถในการปรับปรุงหรือซ่อมแซมทักษะ (Skill) เป็นรายทักษะ โดยไม่ต้องทำการติดตั้งใหม่ทั้งหมด (Full Install) เพื่อความรวดเร็วและลดผลกระทบต่อทักษะอื่นๆ ในกองเรือ

## 2. Scope Consensus (The 4-Layer Logic)

### 2.1 Requirement Mapping (AI Synthesis)
- พัฒนาคำสั่ง `itgo refresh <skill-name>`
- ระบบต้องทำการลบทักษะเดิมใน Agent และคัดลอกไฟล์ใหม่จาก Source/VFS ลงไปแทนที่
- อัปเดตข้อมูลใน Rich Manifest (`.it-skill.json`) เฉพาะรายการที่ถูก Refresh

### 2.2 Information Gathering (Research)
- **IG-1 (Technical)**: ต้องเพิ่มฟังก์ชัน `installSingleSkill` ใน `installer.ts` และลงทะเบียน Sub-command ใน `itgo.ts`
- **IG-2 (Operational)**: ใช้ในกรณีที่ไฟล์ทักษะบางอย่างเสียหาย หรือต้องการอัปเดตเฉพาะฟีเจอร์ใหม่ที่เพิ่ง Forge เสร็จ

### 2.3 Implementation Governance (Layer 4)
- **Execution Skill**: `build-patch`

### 2.4 Pathway (Confirmation)
- **Confirm Command**: **`brfc-Phase 2 confirm`**

## 3. ตรรกะการทำงานหลัก (Core Logic)
1. ตรวจสอบชื่อทักษะที่ระบุ:
   - หากไม่มีใน Source: แจ้งเตือน "Skill not found"
2. ระบุพาธเป้าหมายใน Agent (Local/Global)
3. ดำเนินการลบทักษะเดิม (หากมี)
4. คัดลอกไฟล์ใหม่ (รวมถึงการทำ Stamping version)
5. ปรับปรุงไฟล์ `.it-skill.json` ให้ข้อมูล Version เป็นปัจจุบัน

## 4. มาตรฐานการแสดงผล (Output Standard)
- Log: `♻️  Refreshing [skill-name]...`
- Success: `✅ [skill-name] (vX.Y.Z) refreshed successfully.`
