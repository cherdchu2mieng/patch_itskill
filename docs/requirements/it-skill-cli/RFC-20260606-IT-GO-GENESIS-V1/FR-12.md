# Functional Requirement: Immutable Backups (FR-12)

## 1. Overview
บังคับใช้กฎ "Nothing is Deleted" โดยการสร้างระบบสำรองข้อมูลอัตโนมัติทุกครั้งที่มีการแก้ไขหรืออัปเดตทักษะ เพื่อป้องกันการสูญเสียประวัติการทำงานและ RFC สำคัญ

## 2. Scope Consensus (The 4-Layer Logic)

### 2.1 Requirement Mapping (AI Synthesis)
- พัฒนาระบบ Auto-backup ก่อนกระบวนการเขียนทับใดๆ
- สร้างไฟล์สำรองในรูปแบบ `.bak.<timestamp>`

### 2.2 Information Gathering (Research)
- **IG-1 (Technical)**: ใช้คำสั่ง `mv` หรือ `cp -r` พร้อมผนวก `date +%s` เข้ากับชื่อไฟล์
- **IG-2 (Operational)**: ผู้ใช้สามารถกู้คืนสถานะก่อนหน้าได้เสมอหากการอัปเดตมีปัญหา

### 2.3 Implementation Governance (Layer 4)
- **Execution Skill**: `build-patch`

### 2.4 Pathway (Confirmation)
- **Confirm Command**: **`brfc-Phase 3.12 confirm`**

## 3. ตรรกะการทำงานหลัก (Core Logic)
1. ตรวจสอบว่ามีโฟลเดอร์เป้าหมายอยู่แล้วหรือไม่
2. หากมี ให้ทำการเปลี่ยนชื่อ (Rename) โฟลเดอร์เดิมเป็น `<name>.bak.<timestamp>`
3. ดำเนินการติดตั้งหรืออัปเดตข้อมูลใหม่ลงในพาธเป้าหมาย
4. บันทึกประวัติการสำรองข้อมูลลงใน System Log

## 4. มาตรฐานการแสดงผล (Output Standard)
- ข้อความแจ้งสถานะ: "Backup created: [path].bak.12345678"
- สรุปผลท้ายคำสั่ง: "N items backed up before update."
