# Functional Requirement: Version Auditing (FR-10)

## 1. Overview
ระบบตรวจสอบและแจ้งเตือนเวอร์ชันของเครื่องมือสายคุมกฎ โดยเปรียบเทียบกับเวอร์ชันล่าสุดใน Repository หลักขององค์กร เพื่อความปลอดภัยและทันสมัยของกองเรือ

## 2. Scope Consensus (The 4-Layer Logic)

### 2.1 Requirement Mapping (AI Synthesis)
- ตรวจสอบเวอร์ชันในเครื่อง (`it-skill --version`)
- ดึงข้อมูล Tags ล่าสุดจาก `itinfosv/it-skill-cli` ผ่าน GitHub API
- แจ้งสถานะ `stale` หากพบว่าล้าสมัย

### 2.2 Information Gathering (Research)
- **IG-1 (Technical)**: ใช้ `curl` หรือ GitHub SDK ในการ fetch ข้อมูล release metadata
- **IG-2 (Operational)**: ป้องกันการใช้เครื่องมือคุมกฎเวอร์ชันเก่าที่อาจมีช่องโหว่หรือมาตรฐานที่ไม่เป็นปัจจุบัน

### 2.3 Implementation Governance (Layer 4)
- **Execution Skill**: `build-patch`

### 2.4 Pathway (Confirmation)
- **Confirm Command**: **`brfc-Phase 3.10 confirm`**

## 3. ตรรกะการทำงานหลัก (Core Logic)
1. อ่านค่าเวอร์ชันปัจจุบันจาก `package.json` ของระบบ
2. ส่ง Request ไปยัง GitHub เพื่อตรวจสอบเวอร์ชันล่าสุด
3. ทำการเปรียบเทียบเลขเวอร์ชันตามมาตรฐาน SemVer
4. หากเก่ากว่า ให้ทำเครื่องหมายแจ้งเตือนใน Dashboard

## 4. มาตรฐานการแสดงผล (Output Standard)
- สถานะในตาราง: `⚠️ Stale (Update available: vX.X.X)`
- ข้อความแนะนำ: "Run './install.sh' to update IT-SKILL tools."
