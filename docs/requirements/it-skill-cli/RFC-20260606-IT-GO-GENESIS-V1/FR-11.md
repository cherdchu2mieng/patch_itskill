# Functional Requirement: Governance Tiers (FR-11)

## 1. Overview
การจัดระดับชุดเครื่องมือ (Profiles) สำหรับการติดตั้งและใช้งาน โดยเน้นไปที่ความต้องการของทีม IT เช่น ชุดมาตรฐานสำหรับการคุมกฎทั่วไป และชุดทดลองสำหรับฟีเจอร์ใหม่

## 2. Scope Consensus (The 4-Layer Logic)

### 2.1 Requirement Mapping (AI Synthesis)
- กำหนดกลุ่มทักษะใน `profiles.ts` เฉพาะของ `it-skill-cli`
- สร้าง Profile: `standard` (Governance Essentials) และ `lab` (Experimental Governance)

### 2.2 Information Gathering (Research)
- **IG-1 (Technical)**: ปรับปรุงโครงสร้าง JSON/Object ในการเก็บข้อมูล Profile ให้รองรับการแยกขาดจาก Arra
- **IG-2 (Operational)**: ผู้ใช้สามารถเลือกติดตั้งเฉพาะเครื่องมือที่จำเป็นสำหรับการทำงานแต่ละประเภทได้

### 2.3 Implementation Governance (Layer 4)
- **Execution Skill**: `build-patch`

### 2.4 Pathway (Confirmation)
- **Confirm Command**: **`brfc-Phase 3.11 confirm`**

## 3. ตรรกะการทำงานหลัก (Core Logic)
1. นิยามรายการทักษะที่อยู่ในแต่ละ Profile
2. เมื่อรับคำสั่ง `itgo [profile]` ให้ดึงรายการทักษะที่เกี่ยวข้องขึ้นมา
3. ดำเนินการติดตั้งหรือตรวจสอบเฉพาะทักษะในกลุ่มนั้น
4. รายงานสถานะความครบถ้วนของ Profile ใน Dashboard

## 4. มาตรฐานการแสดงผล (Output Standard)
- สรุปผลท้าย Dashboard: "Governance Profile: standard (4/4 tools installed)"
- ตัวเลือกคำสั่ง: `itgo standard`, `itgo lab`
