# Functional Requirement: itgo Sovereign Manager (FR-13)

## 1. Overview
เปลี่ยนให้ `itgo` เป็นศูนย์กลางการจัดการทักษะ (Sovereign Manager) เพียงหนึ่งเดียว แทนที่คำสั่ง `go` และ `install` เดิม เพื่อสร้างความเป็นระเบียบและอำนาจการควบคุมที่ชัดเจนในระดับองค์กร

## 2. Scope Consensus (The 4-Layer Logic)

### 2.1 Requirement Mapping (AI Synthesis)
- ยุติการทำงานของคำสั่ง `go` (ซึ่งเกิดจาก Default Command ของ `install`)
- ขยายความสามารถของ `itgo` ให้รับ Argument เป็นชื่อโปรไฟล์ (e.g., `itgo standard`)
- พัฒนาระบบดัชนีทักษะแบบละเอียด (Rich Manifest) ที่เก็บเลขเวอร์ชันรายทักษะ

### 2.2 Information Gathering (Research)
- **IG-1 (Technical)**: แก้ไข `commander` options ใน `index.ts` และ `install.ts`. ปรับปรุง `installer.ts` ให้ดึงเวอร์ชันจาก `SKILL.md` ระหว่างติดตั้ง
- **IG-2 (Operational)**: ผู้ใช้จะใช้เพียงคำสั่งเดียวคือ `itgo` สำหรับทุกกิจกรรมการจัดการกองเรือ

### 2.3 Implementation Governance (Layer 4)
- **Execution Skill**: `build-patch`

### 2.4 Pathway (Confirmation)
- **Confirm Command**: **`brfc-Phase 2 confirm`**

## 3. ตรรกะการทำงานหลัก (Core Logic)
1. ตรวจสอบ Argument ที่ส่งมายัง `itgo`:
   - ไม่มี Argument -> แสดง Dashboard (โหมดรวดเร็ว)
   - มี Argument (e.g., `standard`) -> เรียกใช้ลอจิกการติดตั้งโปรไฟล์นั้นๆ
2. ในขั้นตอนการติดตั้ง:
   - สแกนหาลายเซ็นเวอร์ชันใน `SKILL.md` ของแต่ละทักษะ
   - บันทึกข้อมูลลงใน `.it-skill.json` ในรูปแบบ List of Objects
3. ลบ `isDefault: true` ออกจาก `install` command

## 4. มาตรฐานการแสดงผล (Output Standard)
- ข้อความแจ้งเตือนเมื่อพิมพ์ `it-skill go`: "Command 'go' is deprecated. Please use 'itgo' instead." (Optional) หรือให้ระบบแจ้ง "Unknown command" ตามมาตรฐานเพื่อความเด็ดขาด
- Dashboard จะแสดงเลขเวอร์ชันที่ถูกต้องและแม่นยำดึงจาก JSON ดัชนี
